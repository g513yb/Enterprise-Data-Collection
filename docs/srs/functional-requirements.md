# 3 功能需求

## 3.1 企业

### **FR-101 企业信息修改**

企业用户可修改本企业的基础信息。修改完成后需保存并上报至省局备案。企业信息需按照统一模板填写，部分字段不可修改。

**数据说明：**

| 数据项       | 必填 | 说明                                          |
| :----------- | :--- | :-------------------------------------------- |
| 所属地区     | 是   | 显示企业所属地市、市县、区域，不可修改        |
| 组织机构代码 | 是   | 只可输入字母、数字，不超过9位（统一编码规范） |
| 企业名称     | 是   | 中文、英文                                    |
| 企业性质     | 是   | 两级下拉选择                                  |
| 所属行业     | 是   | 两级下拉选择                                  |
| 主要经营业务 | 是   | 按实际情况填写企业主要经营的业务              |
| 联系人       | 是   | 中文、英文                                    |
| 联系地址     | 是   | 两级下拉选择                                  |
| 邮政编码     | 是   | 只可填写6位数字                               |
| 联系电话     | 是   | 格式必须符合（区号）+电话号码或者为手机号码   |
| 传真         | 是   | 格式必须符合（区号）+电话                     |
| EMAIL        | 否   | 格式必须符合xxx@xxx.xxx                       |

**相关功能：**

- 查看当前企业信息
- 编辑可修改字段
- 保存修改并触发备案流程
- 字段格式校验

### **FR-102 备案上报**

企业完成信息填写或修改后，可将备案信息上报至省局。上报后信息进入待审核状态。

**相关功能：**

- 提交备案信息
- 提示提交成功
- 查看备案状态（待审核/已通过/已退回）
- 退回时可查看退回理由

### **FR-103 数据填报**

企业在规定的上报期内，填报本企业的就业失业数据。填报内容需符合模板要求，若调查期就业人数低于建档期人数，则需填写减少类型及原因。

**数据说明：**

| 数据项           | 必填 | 说明                                                         |
| :--------------- | :--- | :----------------------------------------------------------- |
| 建档期就业人数   | 是   | 填写初次建档时监测点就业人数                                 |
| 调查期就业人数   | 是   | 填写本次调查期当时的监测点就业人数                           |
| 其他原因         | 是   | -                                                            |
| 就业人数减少类型 | 否*  | 关闭破产、停业整顿、经济性裁员、业务转移、自然减员、正常解除或终止劳动合同、国际因素变化影响、自然灾害、重大事件影响、其他 |
| 主要原因         | 否*  | 产业结构调整，重大技术改革，节能减排、淘汰落后产能，订单不足，原材料涨价，工资、社保等用工成本上升，自然减员，经营资金困难，税收政策变化，季节性用工，其他，自行离职，工作调动、企业内部调剂，劳动关系转移、劳务派遣 |
| 主要原因说明     | 否*  | -                                                            |
| 次要原因         | 否   | -                                                            |
| 次要原因说明     | 否   | -                                                            |
| 第三原因         | 否   | -                                                            |
| 第三原因说明     | 否   | -                                                            |

*注：当调查期就业人数小于建档期就业人数时，就业人数减少类型、主要原因及主要原因说明为必填*

**相关功能：**

- 填报当期数据
- 保存草稿
- 提交上报
- 校验必填项及逻辑规则（数据对比、格式校验）
- 显示填报期限提醒

### **FR-104 数据查询**

企业用户可查询历史填报数据及其状态（如已上报、已审核、已退回等）。查询结果仅支持浏览，不支持导出。

**相关功能：**

- 按调查期查询
- 按数据状态筛选（草稿/已上报/已审核/已退回）
- 查看数据详情
- 查看审核状态及退回备注

### **FR-105 密码修改**

企业用户可修改登录密码。修改后需重新登录。

**相关功能：**

- 验证旧密码
- 输入新密码并确认（密码复杂度校验）
- 提示修改成功

## 3.2 省

### FR-201 企业备案

**1. 功能描述**

省级用户可查看全省各市已备案企业的详细信息，并对企业备案状态进行审核。审核通过后，企业方可进行数据填报。

**2. 前置条件**

- 用户已登录系统并具备省级角色权限
- 企业已完成基础信息填报并提交备案

**3. 后置条件**

- 企业备案状态更新为“已备案”或“已退回”
- 已备案企业可参与后续数据填报流程

**4. 基本流程**

- 省级用户进入“企业备案”功能模块

- 系统展示企业备案列表，默认按提交时间倒序排列

- 用户可通过调查期、地区、企业名称、备案状态等条件查询企业

- 用户点击企业名称查看详细备案信息

- 用户对备案信息进行审核

  - 若信息无误，点击“审核通过”

  - 若信息有误，点击“退回”，并填写退回原因

- 系统保存审核结果并更新备案状态

**5. 扩展流程**

- 用户可将当前查询结果导出为Excel文件
- 支持批量审核操作（可选）

6. **数据项说明**

| 数据项       | 说明                                   |
| :----------- | :------------------------------------- |
| 所属地区     | 显示企业所属地市、市县、区域，不可修改 |
| 组织机构代码 | 字母/数字，不超过9位                   |
| 企业名称     | 中文/英文                              |
| 企业性质     | 两级下拉选择                           |
| 所属行业     | 两级下拉选择                           |
| 联系人       | 中文/英文                              |
| 联系电话     | 符合（区号）+电话或手机号格式          |
| 备案状态     | 待审核/已备案/已退回                   |
| 审核人       | 系统自动记录当前操作人                 |
| 审核时间     | 系统自动记录                           |

**7. 界面要求**

- 列表页支持分页、排序、条件筛选
- 详情页只读展示企业备案信息
- 审核操作需二次确认

### FR-202 企业查询

**1. 功能描述**

省级用户可根据需要对备案企业进行多条件查询，便于管理与统计。

**2. 前置条件**

- 用户已登录并具备省级角色权限

**3. 后置条件**

- 无

**4. 基本流程**

1. 省级用户进入“企业查询”功能模块
2. 输入或选择查询条件（支持组合查询）
3. 点击“查询”，系统展示符合条件的企列表
4. 用户可点击企业名称查看详细信息
5. 点击“清除”可清空所有查询条件

**5. 扩展流程**

- 支持导出查询结果为Excel文件

**6. 查询条件说明**

| 查询项       | 说明         |
| :----------- | :----------- |
| 企业名称     | 模糊匹配     |
| 组织机构代码 | 精确匹配     |
| 所属地市     | 下拉选择     |
| 所属市县     | 下拉选择     |
| 企业性质     | 下拉选择     |
| 所属行业     | 下拉选择     |
| 备案状态     | 下拉选择     |
| 调查期       | 日期范围选择 |

### FR-203 报表管理

**1. 功能描述**

省级用户对各市审核通过的企业上报数据进行最终审核，并汇总上报至部级单位。

**2. 前置条件**

- 企业数据已通过市级审核
- 用户具备省级审核权限

**3. 后置条件**

- 数据状态更新为“省级审核通过”或“已退回”
- 已上报数据不可再次修改

**4. 基本流程**

1. 省级用户进入“报表管理”模块
2. 系统展示待审核的企业上报数据列表
3. 用户点击企业名称查看详细数据及报表
4. 用户进行审核操作：
   - 数据无误：点击“审核通过”
   - 数据有误：点击“退回修改”，填写退回原因
5. 系统更新数据状态
6. 审核通过后，用户可选择“上报至部委”

**5. 扩展流程**

- 支持批量审核、批量退回
- 上报操作需二次确认

**6. 数据状态说明**

| 状态         | 说明                     |
| :----------- | :----------------------- |
| 待省级审核   | 企业数据已通过市级审核   |
| 省级审核通过 | 省级审核通过，可上报部委 |
| 已退回       | 退回至市级或企业         |
| 已上报       | 数据已上报至部委         |

### FR-204 数据修改

**1. 功能描述**

对于有明显错误的企业上报数据，省级用户有权进行修正。修正后的数据需与原数据区分存储，并记录修改日志。

**2. 前置条件**

- 数据状态为“待省级审核”或“省级审核通过”
- 用户具备数据修改权限

**3. 后置条件**

- 生成修改后的数据副本
- 原始数据保持不变
- 系统记录修改日志

**4. 基本流程**

1. 省级用户进入“数据修改”模块
2. 按调查期、企业名称等条件定位需修改的数据
3. 查看原始数据，点击“修改”
4. 在数据填报界面进行修改
5. 保存修改内容
6. 系统生成修改记录

**5. 扩展流程**

- 支持查看历史修改记录
- 支持导出修改日志

**6. 日志记录字段**

| 字段       | 说明                 |
| :--------- | :------------------- |
| 修改人     | 当前操作人           |
| 修改时间   | 系统自动记录         |
| 修改前内容 | JSON格式或字段级对比 |
| 修改后内容 | JSON格式或字段级对比 |

### FR-205 数据删除

**1. 功能描述**

省级用户可删除历史数据，适用于数据清理或错误数据移除场景。

**2. 前置条件**

- 数据状态为“已上报”或“历史数据”
- 用户具备数据删除权限

**3. 后置条件**

- 数据从系统中移除
- 系统记录删除日志

**4. 基本流程**

1. 省级用户进入“数据删除”模块
2. 按调查期、企业名称等条件选择需删除的数据
3. 点击“删除”，系统弹出二次确认框
4. 用户确认后，数据被移除
5. 系统记录删除操作

**5. 扩展流程**

- 支持批量删除
- 删除前可预览数据内容

**6. 删除日志字段**

| 字段     | 说明             |
| :------- | :--------------- |
| 删除人   | 当前操作人       |
| 删除时间 | 系统自动记录     |
| 删除数据 | 数据摘要或ID列表 |

### FR-206 数据退回

**1. 功能描述**

对于不符合要求的企业上报数据，省级用户可将其退回至上一级（市级或企业），并附上退回原因。

**2. 前置条件**

- 数据状态为“待省级审核”
- 用户具备退回权限

**3. 后置条件**

- 数据状态更新为“已退回”
- 企业端可见退回原因并可重新填报

**4. 基本流程**

1. 省级用户进入“数据退回”模块
2. 选择一条或多条待审核数据
3. 点击“退回”，弹出退回说明填写框
4. 填写退回原因（必填）
5. 确认退回，系统更新数据状态
6. 系统记录退回日志

**5. 扩展流程**

- 支持批量退回
- 退回原因支持模板选择

**6. 退回日志字段**

| 字段     | 说明         |
| :------- | :----------- |
| 退回人   | 当前操作人   |
| 退回时间 | 系统自动记录 |
| 退回原因 | 用户填写内容 |
| 退回数据 | 数据ID列表   |

### FR-207 数据汇总
**1. 功能描述**

省级用户可根据不同的调查期，查看全省范围内企业就业失业数据的汇总统计信息。汇总数据以表格形式展示，便于宏观掌握全省就业岗位变动情况。

**2. 前置条件**

用户已登录系统并具备省级角色权限。

相关调查期内存在已上报的数据。

**3. 后置条件**

无。

**4. 基本流程**

省级用户进入“数据汇总”功能模块。

系统默认展示最新调查期的汇总数据。

用户可通过下拉选择其他调查期，切换汇总数据视图。

系统根据选定的调查期，重新计算并展示汇总信息。

**5. 数据说明**

| 汇总指标	| 说明
| :----------- | :-------------------------------------------- |
|企业总数	|已备案且已填报的企业总数 |
|建档期总岗位数	|所有企业建档期就业人数之和 |
|调查期总岗位数	|所有企业调查期就业人数之和 |
|岗位变化总数	|调查期总岗位数 - 建档期总岗位数 |
|岗位减少总数	|统计所有就业人数减少的企业，其减少人数之和 |
|岗位变化数量占比	|(岗位变化总数 / 建档期总岗位数) * 100% |

**6. 相关功能**

统计： 系统自动计算并展示选定调查期的汇总数据。

导出： 支持将当前汇总表格导出为Excel文件。

### FR-208 数据导出
**1. 功能描述**

省级用户可根据报送期、企业信息、报表类型等条件，将系统中的企业基础信息、企业填报数据等导出为Excel文件，用于离线分析或存档。

**2. 前置条件**

用户已登录系统并具备省级角色权限。

**3. 后置条件**

系统生成并导出Excel文件。

**4. 基本流程**

省级用户进入“数据导出”功能模块。

选择导出类型（如：企业信息、企业报表）。

选择导出数据的调查期（报送期）。

选择或输入其他筛选条件（如：所属地区、企业性质等）。

点击“导出”，系统开始生成文件。

导出完成后，浏览器自动下载文件。

**5. 扩展流程**

若数据量较大，系统可提示“正在后台生成，请稍后”或支持异步导出。

用户可导出当前查询列表（如“企业备案”列表或“报表管理”列表）的结果。

**6. 相关功能**

导出企业信息： 按条件导出企业基础信息和备案状态。

导出企业报表： 按条件导出一家或多家企业在指定调查期的就业失业数据。

无。

点击“查询”，系统展示符合条件的用户列表。

点击“清除”，清空所有已输入的查询条件。

点击“导出”，将当前查询结果导出为Excel文件。

**5. 查询条件说明**

|查询项	|说明
| :------| :-------------------|
|单位名称	|模糊匹配 |

|所处区域	|下拉选择 |
|数据状态	|下拉选择（如：已备案、已上报）|
|单位性质	|下拉选择（企业性质）|
|所属行业	|下拉选择 |

### FR-210 多维分析
**1. 功能描述**

省级用户可通过多维分析功能，从地区、企业性质、行业等多个维度，对全省企业的岗位变动情况进行交叉分析，深入洞察就业市场结构变化。

**2. 前置条件**

用户已登录系统并具备省级角色权限。

存在已上报的数据。

**3. 后置条件**

无。

**4. 基本流程**

省级用户进入“多维分析”功能模块。

选择一个或多个分析维度（如：先选择地区，再选择行业）。

系统根据所选维度，动态生成交叉分析表。

分析表展示不同维度组合下的岗位变化指标。

用户可切换分析指标，如岗位变化总数、岗位变化数量占比等。

**5. 分析维度示例**

|维度	|说明
| :------ | :---------------------- |
|地区	|按州市、区县进行细分|
|企业性质	|按国有、民营、外资等性质划分 |
|所属行业	|按国民经济行业分类进行细分 |
|企业规模	|按人数规模划分（如：0-50人，51-200人） |

**6. 数据说明**

|分析指标	| 说明
| :----------- | :----------------------------- |
|岗位变化总数	| 调查期总岗位数 - 建档期总岗位数 |
|岗位变化数量占比	| (岗位变化总数 / 建档期总岗位数) * 100% |

### FR-211 图表分析

**1. 功能描述**

省级用户可通过折线图、柱状图、表格等形式，直观对比两个不同调查期内的企业岗位变动情况，分析就业形势的变化趋势。

省级用户可通过折线图、表格等形式，展示多个连续调查期内的企业岗位变动趋势，以把握就业市场的长期动态。


**2. 前置条件**

用户已登录系统并具备省级角色权限。

存在至少两个调查期的上报数据。

**3. 后置条件**

无。

**4. 基本流程**

省级用户进入“图表分析” -> “对比分析”子模块。

选择“调查期A”和“调查期B”。

选择分析方式（分析维度）：地区、企业性质或行业。

可选样本条件：按地区、企业性质、行业进行筛选。

点击“查询分析”，系统生成分析结果。

结果以表格和柱状图/折线图两种形式展示对比数据。

省级用户进入“图表分析” -> “趋势分析”子模块。

选择一个连续的时间段（如：2023年第一季度至2024年第一季度）。

选择分析维度（如：总体趋势、按地区、按企业性质）。

点击“查询分析”，系统生成分析结果。

结果以折线图和表格形式展示，X轴为调查期，Y轴为分析指标。

**5. 数据说明**

|分析指标	|说明
| :-------- | :-------------------------------------- |
|企业总数	| 两个调查期内均存在的企业总数（可比样本）|
|建档期总岗位数	| 在调查期A的建档期（初始期）的岗位数 |
|调查期总岗位数	| 在调查期B的岗位数 |
|岗位变化总数	| 调查期总岗位数 - 建档期总岗位数 |
|岗位减少总数	| 统计从调查期A到调查期B期间，就业人数减少的企业，其减少人数之和 |
|岗位变化数量占比	| (岗位变化总数 / 建档期总岗位数) * 100% |

|分析指标	|说明
|:-----------|:-----------------------------|
|岗位变化数量占比	|(岗位变化总数 / 建档期总岗位数) * 100% |

### FR-212 发布通知
**1. 功能描述**

省级用户可向全省或指定范围内的用户发布、删除通知信息，用于发布政策、公告、填报提醒等。

**2. 前置条件**

用户已登录系统并具备省级角色权限。

**3. 后置条件**

新增/修改/删除的通知在目标用户端生效。

**4. 基本流程**

列表： 进入“发布通知”模块，系统默认展示当前用户发布的所有通知列表，包含标题、发布时间、发布单位等信息。

新增： 点击“新增”，填写通知标题、内容后发布。

修改： 选择一条通知，进行修改后保存。

删除： 选择一条通知，确认后删除。

**5. 数据说明**

|数据项	|必填	|说明
| :------ | :---- | :--------- |
|通知标题	| 是	| 50字以内 |
|通知内容	| 是	| 2000字以内 |
|发布时间	| 否	| 系统自动生成 |
|发布单位	| 否	| 系统自动生成 |

**6. 相关功能**

列表： 展示所有已发布的通知，支持按标题、时间排序。

新增： 发布新通知。

修改： 修改通知内容。

删除： 删除通知，删除后用户端不可见。

发布范围： （可选）可设定通知的接收范围，如全省、指定地市。

### FR-214 系统管理

**1. 功能描述**

省级用户可新增或修改调查期，设定企业数据填报的开始与截止时间。

省级用户可创建、管理省本级和企业的用户账号，并为用户分配角色。

省级用户可根据管理需要，定义和修改角色，并为不同角色分配不同的功能权限。系统预定义一些基础角色。

省级用户可查看当前系统的运行情况，包括服务器CPU、内存、硬盘等硬件资源信息，以及系统运行状态、用户活跃度等应用信息。

**2. 前置条件**

用户已登录系统并具备省级角色权限。

省级用户可创建、管理省本级和企业的用户账号，并为用户分配角色。

用户已登录系统并具备省级角色权限。

**3. 后置条件**

新的上报时限对全省企业和各级管理部门生效。

新增/修改/删除的用户信息生效。

新增/修改/删除的角色及其权限配置生效。

**4. 基本流程**

进入“系统管理” -> “上报时限”子模块。

新增： 点击“新增”，填写调查期名称（如“2024年第一季度”）、填报开始日期、填报结束日期，确认后保存。

修改： 在列表中，选择一个调查期，修改其填报开始/结束日期，确认后保存。

删除： 对于未开始或已过期的调查期，可选择删除。

进入“系统管理” -> “用户管理”子模块，查看所有用户列表。

新增： 点击“新增”，填写用户信息（账号、密码、姓名、所属单位、联系电话等），并为其分配一个或多个角色。

修改： 在列表中选择一个用户，修改其基本信息或角色。

删除： 在列表中选择一个用户进行删除。若该用户已有上报数据，系统应提示“该用户已有上报数据，无法删除”，如需删除需先处理其关联数据。

进入“系统管理” -> “角色管理”子模块，查看所有角色列表。

角色定义： 点击“新增角色”，填写角色名称，然后在权限树中勾选该角色可访问的功能模块，保存后即创建一个新角色。

角色修改： 在列表中选择一个角色，修改其权限分配。

角色删除： 在列表中选择一个角色进行删除。若该角色已分配给用户，系统应提示“该角色已分配给用户，删除后需为相关用户重新分配角色”，确认后删除角色及其与用户的关联关系。

进入“系统管理” -> “系统监控”子模块。

系统以仪表盘、图表等形式展示各项监控指标。

用户可查看实时或近期的监控数据。

**5. 数据说明**

|数据项	| 说明
| :-------- | :-------------------------------------- |
|调查期名称	|如“2024年第一季度” |
|开始日期	|企业可开始填报的日期 |
|结束日期	|企业必须完成填报的日期 |


|数据项	| 说明
| :-------- | :-------------------------------------- |
|登录账号	| 唯一标识，不可重复|
|用户姓名	| 中文、英文|
|用户类型	| 省用户/市用户/企业用户|
|所属单位	| 关联的单位信息|
|联系电话	| 手机号码|
|角色	| 从角色库中选择|
|状态	| 启用/禁用|

|监控项	|说明
| :-------- | :-------------------------------------- |
|硬件信息 | |	
|CPU 使用率	| 当前服务器的CPU占用百分比 |
|内存使用率	| 当前服务器的内存占用百分比 |
|硬盘使用率	| 当前服务器的硬盘空间占用情况 |
|应用信息 | |	
|在线用户数	|当前登录系统的用户数量 |
|活跃会话数	|系统当前活跃的会话数量 |
|接口响应时间	| 关键业务接口的平均响应时间 |
|错误日志数	| 近期系统产生的错误日志数量 |

## 3.3 市

### **FR-301 数据审核**

市级用户审核辖区内企业上报的就业失业数据。审核通过后数据可上报至省级。

**数据说明：**

审核内容为企业填报数据（详见FR-103数据说明）

**相关功能：**

- 列表展示待审核数据（分页显示，可按企业名称、调查期筛选）
- 查看企业填报详情（包含历史填报记录）
- 审核通过/退回（退回时需填写退回理由）
- 批量审核功能
- 查看审核记录（审核人、审核时间、审核结果）

### **FR-302 通知发布**

市级用户可向辖区内企业发布通知。通知内容需包含标题、正文、发布时间等信息。

**数据说明：**

| 数据项   | 必填 | 说明         |
| :------- | :--- | :----------- |
| 通知标题 | 是   | 50字以内     |
| 通知内容 | 是   | 2000字以内   |
| 发布时间 | 否   | 系统自动生成 |
| 发布单位 | 否   | 系统自动生成 |

**相关功能：**

- 新增通知（填写标题、内容）
- 修改通知（仅可修改未过期的通知）
- 删除通知
- 查看已发布通知列表（按发布时间倒序）
- 通知阅读统计（可选）

### **FR-303 密码修改**

市级用户可修改登录密码。修改后需重新登录。

**相关功能：**

- 验证旧密码
- 输入新密码并确认（密码复杂度校验：长度、字符类型）
- 提示修改成功
- 记录密码修改日志

## 2.4 运行环境

本系统采用 B/S 架构，用户通过浏览器访问，服务器集中部署与运维。

- 客户端：支持主流现代浏览器（Chrome、Edge、Firefox、Safari 及基于 Chromium 的国产浏览器），无需安装专用客户端；建议分辨率不低于 1366×768；具备稳定网络连接。
- 服务器端：建议部署于政务云或私有云环境；采用 64 位 Linux 系统；应用服务与数据库分离部署并支持备份与容灾；具备负载均衡与扩展能力。
- 网络与安全：支持政务外网与互联网访问场景；全链路使用 HTTPS；具备防火墙/WAF 等基础安全防护能力。

## 2.5 设计与实现约束

为保障合规性、可维护性与可扩展性，系统在设计与实现上需满足以下约束：

- 合规约束：遵循国家与云南省政务数据管理、安全与保密相关法律法规及标准要求，满足等保要求。
- 数据标准约束：企业唯一标识、地区与行业等基础编码遵循国家/省级发布的数据字典与编码标准；时间日期与统计口径保持一致。
- 审计与留痕约束：对企业上报数据的修正需保留原始记录并记录修改人、时间与变更内容，确保可追溯。
- 时效性约束：数据填报、审核与汇总需受调查期/上报期控制，到期自动锁定相关入口。
- 技术约束：接口采用标准化的 REST/JSON 风格，前后端分离实现，支持后续与外部系统对接与扩展。

## 2.6 假设与依赖

系统顺利实施依赖以下前提条件：

- 外部系统依赖：如需与国家失业监测系统进行数据交换，依赖其接口规范、联调窗口与稳定可用的服务能力。
- 基础数据依赖：企业名录、地区编码、行业分类与指标口径等由业务主管部门提供并持续维护。
- 网络与设施依赖：企业端具备稳定互联网访问条件，市/省级用户具备稳定的政务网访问条件。
- 人员与组织假设：相关单位安排专人填报与审核，具备基础计算机操作能力并接受必要培训。
